everything you need to know about root_chat

getting started

On first launch, root_chat connects you to the public relay in the public room. You can chat with anyone there, or switch to a private room or your own relay server.

change your username

/name yourname

join a different room

/room myroom

Rooms are created automatically — pick any name for a private space. Share the name with others to meet there.

commands

/room <name>switch to a different room
/roomslist all active rooms on the relay
/name <newname>change your username
/dm <user>open a private chat with someone
/msg <user> <text>send a one-off private message
/add <username>add someone to contacts
/remove <username>remove from contacts
/connectlist saved connections
/connect newadd a new connection
/connect edit <n>edit saved connection n
/connect delete <n>delete saved connection n
/mutemute notifications
/unmutere-enable notifications
/autostarttoggle launch on system login
/updatecheck for and apply updates
/restartrestart root_chat
/clearclear the message display
/helpshow all commands
/quitexit

private messaging

Start a private conversation with another user in the room:

/dm alice
· both users land in a private room · only you two can see it

Send a quick one-off message without switching rooms:

/msg alice hey, are you free?

using your own relay server

To connect to a different relay (your own or someone else's), add a new connection:

/connect new
· follow the setup wizard
· select mode: relay
· enter your server URL, room, and optional key

Switch between saved connections at any time:

/connect list saved connections
/connect 2 switch to connection 2

self-hosting a relay server

The relay server is a lightweight FastAPI app. Run it on any Linux VPS or home server.

quick setup (Ubuntu / Debian)

# clone and set up
$ git clone https://github.com/r3pc0n/root_chat.git
$ cd root_chat
$ bash server/setup.sh

optional: require an API key

Create a .env file next to relay.py:

RELAY_API_KEY=your-secret-key

Users connect with: /connect new → enter the key in the "relay key" field.

TLS with Caddy

relay.yourdomain.com {
reverse_proxy localhost:7332
}

Caddy handles TLS automatically. Users connect via wss://relay.yourdomain.com.

encryption

direct mode (host / connect)

End-to-end encrypted automatically. X25519 key exchange + ChaCha20-Poly1305. Your fingerprint is shown on connect — verify it with your peer out-of-band.

relay mode

The relay server sees who is in each room but cannot read messages when end-to-end encryption is enabled. Set a message encryption key in the connection wizard ("message enc key") — both sides need the same key.

  • Key is derived with PBKDF2-HMAC-SHA256 (100k iterations)
  • Same key in different rooms produces different encryption keys
  • System messages (join/leave) are not encrypted
